Easy — XOR warmup

Intended path: recognize that strings -a shows nothing useful, drop into objdump -d crackme-easy, find the verify loop with a literal xor 0x42 on each byte, dump .rodata with objdump -s -j .rodata, and xor the bytes against 0x42 in Python.

What's in the binary:

const unsigned char enc[] = { /* flag XOR'd against 0x42 */ };
for (size_t i = 0; i < enc_len; i++) {
    if (input[i] != (unsigned char)(enc[i] ^ 0x42)) return 0;
}

Why strings fails you here: the printable bytes of the flag, after XOR with 0x42, become non-printable garbage. strings only prints runs of printable ASCII, so it skips the array entirely. strings -a won't help — there's nothing printable to find. Lesson: strings is a smoke test, not a solve.

One-liner:

$ objdump -s -j .rodata crackme-easy | python3 -c '
import sys, re
blob = bytes.fromhex("".join(re.findall(r"[0-9a-f]{2}", sys.stdin.read())))
for s in range(len(blob)):
    decoded = bytes(b ^ 0x42 for b in blob[s:])
    m = re.search(rb"m190\{[A-Za-z0-9_]{8,40}\}", decoded)
    if m: print(m.group().decode()); break'

Why this tier exists: establishes a baseline. Disassembly literacy + recognizing trivial obfuscation. Anyone who's done a few crackmes will breeze through; folks new to RE will spend 10–20 min and learn something. Medium starts assuming you got here.
