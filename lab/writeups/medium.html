Medium — anti-debug + per-byte transform

Intended path:

  1. Spot the ptrace trap. The first thing the binary does is ptrace(PTRACE_TRACEME, 0, 0, 0). If you started in gdb, ptrace returns -1 and the program exits silently. Set a breakpoint on main, step once, see ptrace, neutralize.
  2. Neutralize. Three options, pick your favorite:
    • Patch the binary: replace the jl/js after the ptrace check with jmp, or NOP the whole syscall.
    • LD_PRELOAD a stub that always returns 0: 
      $ cat > ptrace_stub.c << 'EOF'
long ptrace(int r, int p, void *a, void *d) { return 0; }
EOF
$ gcc -shared -fPIC ptrace_stub.c -o stub.so
$ LD_PRELOAD=./stub.so gdb ./crackme-medium
    • Use strace -e trace=ptrace to confirm the call, then patch.
  3. Reverse the transform. Per-byte: enc[i] = (rotl(input[i], 3) ^ key[i%8]) - i. Inverse: input[i] = rotr((enc[i] + i) ^ key[i%8], 3). The key string lives in .rodata as "m190labs".

Solver:

def rotr8(x, n): return ((x >> n) | (x << (8 - n))) & 0xFF
KEY = b"m190labs"
flag = bytes(rotr8(((enc[i] + i) & 0xFF) ^ KEY[i % 8], 3) for i in range(len(enc)))

Real-world tie-in: per-byte algorithmic flag checks are still the most common shape of beginner-to-intermediate malware string protection. The ptrace self-check is a real-world habit that real malware uses — it's not theoretical, it's just the easiest tier of anti-debug. More on anti-RE techniques in the wild.

If you want to make it harder for yourself: solve it without neutralizing the ptrace check — pure static disassembly + python. Doable.
